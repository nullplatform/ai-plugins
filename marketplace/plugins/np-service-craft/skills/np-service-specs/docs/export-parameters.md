# Parameter Export (Env Vars for Apps)

When a link is activated, NP reads the `attributes` of the service and link, and creates env vars (parameters) in the linked application. Only fields marked with `export: true` or `export: {secret: true}` are exported.

## How It Works

1. **Service spec** defines output fields with `"export": true` (e.g., `bucket_arn`, `bucket_region`)
2. **Create/update workflow** executes `write_service_outputs` post-tofu → updates `service.attributes` via `np service patch`
3. **Link spec** defines credential fields with `"export": true` or `"export": {"type": "environment_variable", "secret": true}`
4. **Link workflow** executes `write_link_outputs` post-tofu → updates `link.attributes` via `np link patch`
5. **When the link activates**, NP combines the exported attributes and creates parameters in the app

## Env Var Naming

Pattern: `{LINK_SLUG_UPPER}_{ATTRIBUTE_NAME_UPPER}`

| Link slug | Attribute | Env var |
|-----------|-----------|---------|
| `my-connection` | `bucket_name` | `MYCONNECTION_BUCKET_NAME` |
| `test-con-params` | `bucket_arn` | `TESTCONPARAMS_BUCKET_ARN` |
| `linktest` | `secret_access_key` | `LINKTEST_SECRET_ACCESS_KEY` |

> The hyphen in the slug is **removed** when converting to upper case (not replaced with `_`).

## Key Behaviors

- **Empty fields don't generate parameters**: If `kms_key_arn` remains empty, `MYLINK_KMS_KEY_ARN` is not created
- **Secrets hidden in API**: `export: {secret: true}` makes the value `null` in API responses. This is normal.
- **Read-only parameters**: Parameters generated by export are immutable from the UI

## E2E Verification

```bash
# 1. Verify service attributes (post create)
/np-api fetch-api "/service/<service_id>"
# .attributes should contain output fields (bucket_arn, etc)

# 2. Verify link attributes (post link)
/np-api fetch-api "/link/<link_id>"
# .attributes should contain credentials
# Fields with export:{secret:true} appear as null (normal)

# 3. Verify parameters in the app
# UI: App → Parameters → search for env vars with pattern {LINK_SLUG_UPPER}_{FIELD_UPPER}
```
